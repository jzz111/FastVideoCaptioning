# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) for Fast Video Captioning.

An ADR records a meaningful technical or product-architecture decision so the project can remember why the choice was made.

## Directory Map

```text
adr/
`-- mvp/
    `-- decisions that apply to the MVP
```

Keep version- or milestone-specific ADRs in the matching subdirectory. Add a new subdirectory when a later release needs decisions that should not automatically apply to the MVP.

## ADR Shape

Each ADR should include:

- Title
- Status
- Date
- Requirement references, when applicable
- Context
- Decision
- Consequences

Use `Accepted` for decisions that guide current implementation. Use `Proposed` when a decision needs more validation before implementation.
