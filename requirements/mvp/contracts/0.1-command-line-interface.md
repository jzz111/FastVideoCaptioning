# Contract 0.1: Command Line Interface

## Requirement References

- `0.1.1` - Display help information
- `0.1.2` - Validate command line arguments
- `0.1.3` - Execute caption-generation workflow
- `0.1.4` - Execute model-management workflow

## Purpose

Epic `0` defines one connected command line interface for the MVP. The CLI supports two workflows:

1. Caption generation.
2. Model management.

The stories in this epic should be tested and implemented story by story, but their contract should be considered together because they share the same arguments, validation rules, model names, defaults, and dispatch behavior.

## Public Source Contract

The source contract lives under `src/cli`.

The primary callable is:

```python
run_cli(args, caption_workflow=None, model_manager=None) -> CliResult
```

`args` is a sequence of command line arguments without the executable name.

`caption_workflow` is an optional callable used by tests and application wiring. It receives a `CaptionGenerationRequest`.

`model_manager` is an optional object used by tests and application wiring. It may provide:

- `list_downloaded_models() -> list[str]`
- `download_model(model_name: str) -> None`

`CliResult` contains:

- `exit_code`
- `stdout`
- `stderr`

The CLI should return structured results instead of writing directly to the console. A future process entry point may translate those results to real stdout, stderr, and process exit codes.

CLI request and result models live under `src/cli/models`. CLI workflow dependencies and external-facing protocols live under `src/cli/ports`. CLI parsing, validation, help rendering, and dispatch behavior live under `src/cli/services`. The top-level `src/cli/command_line.py` module is the CLI orchestration entry point, not a container for every CLI class or helper behavior.

## Supported Arguments

| Argument | Workflow | Required | Description |
| --- | --- | --- | --- |
| `--help`, `-h` | All | No | Display CLI help and exit successfully. |
| `--input` | Caption generation | Yes | Source video or audio file. |
| `--markers` | Caption generation | Yes | Marker export file for YouTube timeline chapters. |
| `--output` | Caption generation | Yes | Clean caption `.txt` output path. |
| `--model` | Caption generation | No | Transcription model to use. |
| `--vocabulary` | Caption generation | No | Channel-specific vocabulary correction file. |
| `--blacklist` | Caption generation | No | Blacklist vocabulary file. |
| `--manage-models` | Model management | Yes | Enter model-management mode. |
| `--download-model` | Model management | No | Select a supported model for download. Requires `--manage-models`. |

## Supported Models

| Model Name | Role |
| --- | --- |
| `whisper-large-v3` | Default captioning model. |
| `distil-whisper` | Alternate captioning model. |

Help text should explain these names using the product names OpenAI Whisper Large-v3 and OpenAI Distil-Whisper.

## Default File Locations

When optional caption-generation file arguments are not provided, the CLI uses:

| Argument | Default |
| --- | --- |
| `--vocabulary` | `data/vocabulary/corrections.csv` |
| `--blacklist` | `data/vocabulary/blacklist.csv` |

The default transcription model is `whisper-large-v3`.

## Validation Rules

Help mode:

- `--help` and `-h` exit successfully.
- Help mode does not require any other arguments.
- Help mode does not start caption generation or model management.

Caption-generation mode:

- Requires `--input`.
- Requires `--markers`.
- Requires `--output`.
- Accepts `--model`, `--vocabulary`, and `--blacklist`.
- Rejects unsupported model names.
- Starts caption-generation processing only after successful validation.

Model-management mode:

- Requires `--manage-models`.
- May accept `--download-model`.
- Rejects unsupported model names for `--download-model`.
- Does not require caption-generation arguments.
- Rejects caption-generation arguments while `--manage-models` is active.
- Prevents caption-generation processing while `--manage-models` is active.

Invalid combinations:

- `--download-model` without `--manage-models` is invalid.
- Calling the CLI with no arguments is invalid.
- Missing required caption-generation arguments produce user-friendly validation messages.
- Validation failures return a non-zero exit code and do not call workflow dependencies.

## Dispatch Rules

After successful caption-generation validation, the CLI calls `caption_workflow` with a `CaptionGenerationRequest` containing:

- `input_path`
- `markers_path`
- `output_path`
- `model`
- `vocabulary_path`
- `blacklist_path`

After successful model-management validation, the CLI uses `model_manager` to:

- Display all supported models.
- Display locally downloaded models.
- Download the requested model when `--download-model` is provided.

If a workflow dependency is missing after successful validation, the CLI returns a user-friendly error instead of silently succeeding.
